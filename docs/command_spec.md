# USB CDC コマンド仕様 — 2ch 位相差計

**対象:** phase_meter_firm_v0
**更新日:** 2026-07-05

ファーム（STM32）と PC GUI が共有する通信仕様。

> GUI の自動計測スクリプト機能は本仕様の既存コマンドのみを使用し、ファーム変更は不要。
> スクリプト文法・保存形式は [`auto_measure.md`](auto_measure.md) を参照。

---

## 1. 通信仕様

| 項目 | 値 |
|---|---|
| インタフェース | USB CDC (Virtual COM Port) |
| 終端文字（送受信） | `\n` (LF, 0x0A) |
| エンコーディング | ASCII |
| コマンドチェーン区切り | `;` |

1 行 1 コマンド（またはチェーン）を送信し、ファームは 1 行で応答する。

---

## 2. 応答フォーマット

| 種類 | 応答 | 説明 |
|---|---|---|
| 成功（値なし） | `OK\n` | 動作コマンド |
| 成功（値あり） | `<値>\n` | クエリコマンド |
| 複数行終端 | `END\n` | `RA` の最終行 |
| 取得完了通知 | `DONE\n` | `RUNN<n>` の指定点数を取得し自動停止した直後に 1 回送出 |
| エラー | `NG\n` | パラメータ異常・実行不可 |

---

## 3. コマンド一覧

| コマンド | 説明 | 応答 |
|---|---|---|
| `RUN` | START へ。位相差を 100 Hz 連続ストリーミング開始（フリーラン） | `OK` |
| `RUNN<n>` | 位相サンプルを `n` 点取得したら自動停止（時間指定取得）。`n`≥1 | `OK`/`NG` |
| `STOP` | WAIT へ。ストリーミング停止 | `OK` |
| `ATTT<dB>` | TX アッテネータ設定 [dB]（例 `ATTT12.5`） | `OK`/`NG` |
| `ATTR<dB>` | RX アッテネータ設定 [dB] | `OK`/`NG` |
| `POTT<v>` | TX ポテンショメータ(MCP41HV51) ワイパ設定 0..256（SPI2/PB3） | `OK`/`NG` |
| `POTR<v>` | RX ポテンショメータ(MCP41HV51) ワイパ設定 0..256（SPI3/PC14） | `OK`/`NG` |
| `TEMP` | TMP126 ×3 の温度を単発読出し [℃] | `<t1>,<t2>,<t3>`（各4桁小数） |
| `TID` | TMP126 ×3 のデバイスID読出し（正常=`2126`） | `<id1>,<id2>,<id3>`（16進4桁） |
| `R11S0`/`R11S1` | trig_src: 0=内部(TIM6) / 1=外部(TIM1 ETR=PE7 の外部クロック) | `OK` |
| `R12S0`/`R12S1` | trig_edge: ETR 極性 0=立上がり / 1=立下がり | `OK` |
| `R<addr>S<val>` | parm レジスタ書込み（焼込みなし） | `OK`/`NG` |
| `R<addr>R` | parm レジスタ読出し | `<値>`/`NG` |
| `RA` | 全 parm 読出し（複数行 + `END`） | `<addr>:<name>:<unit>:<val>` × N |
| `RS` | parm をフラッシュ保存（オプション/フェーズ2） | `OK`/`NG` |
| `VER` | ファーム版数 | `<str>` |
| `WAVE` | 生波形 1 回ダンプ（フェーズ2, 後日） | （後日） |

`<unit>`: 0=int, 1=float, 2=char

---

## 4. ストリーミング出力（START 状態）

ファームは取得・算出ごとに 1 行を送る:

```
F,<seq>,<phase_diff_deg>,<peak_amp_v>,<peak_freq_hz>\r\n
```

例:
```
F,1023,-12.345,0.482156,100000.0
F,1024,-12.301,0.481902,100000.0
```

- `<seq>`: 100 Hz ティック連番（10 ms 刻み）。**時間軸 = seq × 10 ms** とすることで、
  USB 受信のジッタ/バーストに依らず等間隔でプロットできる。
- `<phase_diff_deg>`: ch1 基準の ch2 に対する位相差 [deg]、範囲 (-180, 180]。
- `<peak_amp_v>`: ch1 振幅 [V]（16bit, Vref=3.3V, 小数6桁）。ピーク周辺帯域
  `kpeak±band_w` の**パワー和**から `A=2√(Σ|X₁|²)/N` で算出（漏れ/スキャロップに頑健）。
- `<peak_freq_hz>`: ピーク周波数 [Hz]（`kpeak × fs / N`）。
- 約 100 Hz（10 ms 周期）で出力。GUI は seq を時間軸に deg/amp をグラフ、freq をテキスト表示。
- `RUN` はフリーラン（`STOP` まで無限）。`RUNN<n>` は `F,` を `n` 行出力後に自動停止し、
  最後に `DONE` を 1 行送る（以降 `F,`/`T,` は止まる）。GUI 側は「秒 → 点数（秒×100）」または
  「点数を直接指定」して `RUNN` を送る。

### 4.1 温度ストリーム（START 状態, TMP126 ×3）

START 中は、位相ストリームと並行して TMP126 温度を **10 Hz**（`F,` の 1/10）で送る:

```
T,<tick>,<t1_c>,<t2_c>,<t3_c>\r\n
```

例:
```
T,1030,25.1875,25.2500,24.9375
T,1040,25.1875,25.2812,24.9375
```

- `<tick>`: 100 Hz ティック連番。`F,` の `<seq>` と**同一の時間基準**（`dbg_tick`）。
  時間軸 = tick × 10 ms。GUI は `F,` と同じ時間軸に温度をプロットできる。
- `<t1_c>`/`<t2_c>`/`<t3_c>`: TMP126 #1/#2/#3 の温度 [℃]（小数4桁）。
  #1=SPI2/CS PB1, #2=SPI2/CS PB2, #3=SPI3/CS PC13。
- 送出間引きは `TMP126_STREAM_DIV`（既定 10 = 100 Hz/10）。読出しエラー時はその ch の
  直近有効値を再送する。
- `STOP` でストリーム停止。`TMP126_ENABLED=0`（ファームビルド時）で温度機能を無効化可能。

> GUI は `F,` / `T,` で始まる行をそれぞれ解析して数値を取り出す。`OK`/`NG`/`END` 等の
> 応答行とストリーム行が混在し得るため、プレフィックスで判別すること。

---

## 5. 送信例

```
RUN\n                 → 計測開始（フリーラン, 以後 F,... が連続して届く）
RUNN300\n             → 300 点(≒3 秒)取得後に自動停止 → OK …（F,×300）… DONE
STOP\n                → 計測停止 → OK
ATTT10;ATTR3.5\n      → TX=10dB, RX=3.5dB → OK
R1R\n                 → adc_num 読出し → 4096
R3S100000\n           → target_hz=100000 設定 → OK
VER\n                 → phase_meter_firm_v0
TEMP\n                → 温度3ch 単発読出し → 25.1875,25.2500,24.9375
TID\n                 → デバイスID確認 → 2126,2126,2126（正常）
```

## 6. エラー応答

| 条件 | 応答 |
|---|---|
| 不明なコマンド | `NG` |
| レジスタアドレス範囲外 | `NG` |
| dB が範囲外（HMC8073 の範囲外） | `NG` |
